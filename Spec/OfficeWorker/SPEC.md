# Office RPG Match-3 Game Specification

## 1. 遊戲核心機制 (Core Mechanics)
- **基礎消除 (消除 = 攻擊)**：經典三消玩法，玩家在「消除區」滑動相鄰的方塊（員工日常元素）達成三個以上連線，消除的方塊會化為特定的「職場怨氣武器」飛向敵人（Boss）造成傷害或對玩家產生治癒等效果。
- **Boss 戰鬥系統**：
  - **精神值 (HP) 系統**：玩家與 Boss 皆有生命值。玩家的 HP 代表「精神崩潰邊緣值」，被扣完則遊戲失敗（進入爆肝過勞狀態）；敵人的 HP 扣完則關卡勝利。某一方歸零時觸發勝負。
  - **回合制與行動倒數 (Timer)**：玩家每移動一次方塊算一回合（連鎖不額外扣回合）。Boss 有攻擊倒數（例如：3回合後發動攻擊），玩家每移動一步倒數便減少，倒數歸零時 Boss 發動攻擊，造成玩家傷害並重置倒數。玩家必須在時限內消滅敵人或利用補血方塊續命。
- **掉落與重整 (Gravity & Refill)**：
  - 消除方塊後，上方塊受到重力影響自然落下，並從頂部隨機補充新方塊。
  - 這些連續的掉落可能引發後續的連鎖反應（Combo）。
- **死局洗牌 (Shuffle)**：
  - 當畫面上沒有任何可移動消除的組合時，系統將自動洗牌（Shuffle）。
  - 洗牌機制：將「盤面上現有的方塊（包含特殊道具）」打亂並重新排列，而非重新生成全新方塊。系統會嘗試找出不會造成自動連線且至少有一步可移動的排列；若極端情況下多次重排皆無解，才會作為安全機制重新隨機生成。
## 2. 特殊道具與生成規則 (Special Props)
當玩家完成特定的連線形狀時，會將連線轉化為一個強力的特殊道具，並留在盤面上供後續引爆。
1. **水平/垂直光束 (↔️↕️ 職場急件蓋章)**：
   - 生成條件：水平或垂直 4 連線。
   - 效果：變成發光的「URGENT」印章，引爆時引發橫向或縱向整排貫穿傷害，清除該方塊所在的**整行或整列**。
2. **十字炸彈 (💥 薪水小偷摸魚彈)**：
   - 生成條件：完成 L 型或 T 型的連線。
   - 效果：變成網頁購物車，引爆時引發周圍九宮格大爆炸，清除以自身為中心的一個大區塊。
   - **優化現狀**：生成點已經過嚴格計算，必定精準產生於 L/T 型連線的「交會點（轉角處）」。
3. **彩色黑洞 (🌀 準時下班打卡機)**：
   - 生成條件：完成 5 連線。
   - 效果：變成金色打卡機。與任意顏色方塊交換時，觸發全螢幕大絕「**準時下班！**」，將畫面中所有該顏色的方塊一次全部吸入化為離職信並消除。
   - **視覺現狀**：圖示為一個旋轉發光的黑洞。消除發動時不使用刺眼反白，而是在每個目標方塊下方產生小黑洞，並跟著方塊同步縮小消失（動畫延遲 500ms，讓玩家看清吸收過程）。

## 3. 道具組合技 (Prop Combos)
當玩家「直接將兩個特殊道具互相交換」時，不再觸發普通的連線消除，而是直接引發威力驚人的組合技：
- **光束 + 光束**：產生十字型光束轟炸，清除一整列與一整行。
- **光束 + 十字**：產生「巨型光束轟炸」，破壞範圍為寬度 3 的十字（清除 3 行 + 3 列）。
- **十字 + 十字**：產生「超大核爆」，範圍擴大至 5x5 的巨大區塊。
- **黑洞 + 一般道具 (光束或十字)**：
  - 效果：會將畫面上所有與「被交換的道具」同色的普通方塊，全部轉化為該道具（例如滿畫面的十字炸彈），然後在一瞬間全部連鎖引爆！
- **黑洞 + 黑洞**：
  - 效果：盤面全清。
  - **視覺現狀**：會在畫面正中央生成一個「覆蓋整個遊戲盤面的超巨大黑洞」，接著整個畫面的方塊會跟著大黑洞一起在 1 秒內坍縮消失，然後才降下新方塊。

## 4. 戰鬥數值與音效表現 (Combat & Audio)
- **連擊加乘 (Combo Multiplier)**：
  - Combo 2：1.2x 傷害/回復
  - Combo 3：1.5x 傷害/回復
  - Combo 4 以上：2.0x 傷害/回復
  - 組合技 (Prop Combos) 觸發時，其造成的巨量傷害固定享有 2.0x 的終極加成，且範圍內每消除一顆 KPI（破甲）方塊會再額外 +0.5x 倍率。
  - 多色同時引爆也會再疊加 1.5x 係數。
- **Web Audio 系統**：
  - 揚棄傳統 8-bit 音效，實作了基於 Web Audio API 的進階合成器。
  - 運用 ADSR (Attack, Decay, Sustain, Release) 包絡線、雙振盪器 (Oscillators) 與低通濾波 (Low-pass Filter)，創造出具有 Q 彈感、空間感與和聲的滑順打擊音效。

## 5. 近期技術修正備忘 (Recent Fixes)
1. **API 名稱統一**：修復了 `processPropCombo` 中呼叫到不存在的 `playerAttack` / `stunBoss` 等函式，現已統一對接為 `this.gameState.applyDamage()` 等標準介面。
2. **重力掉落邏輯重構**：將 `processMatches` 中的重力與填充邏輯抽離成獨立的 `applyGravityAndRefill()`，修復了組合技執行後不會掉落方塊的 Bug。
3. **黑洞延遲調校**：黑洞的消除動畫等待時間由 1000ms 縮短至 500ms，在「看清消除過程」與「遊戲流暢節奏」之間取得最佳平衡。
4. **拖曳體驗優化 (Auto-Release)**：在 `MatchEngine` 的 `onPointerMove` 中實作了拖曳自動釋放機制。當玩家拖曳方塊超過本身寬度的一半（50%）時，系統會自動觸發 `onPointerUp` 進行交換，有效解決偶發滑鼠未釋放而卡住的問題，讓操作手感更為流暢。

## 6. 八大基礎方塊設定 (Base Blocks)
遊戲目前實作了 8 種不同的基礎方塊，隨關卡進度逐漸解鎖：
1. **🔴 郵件 (MAIL)**：基礎攻擊 (10 dmg)。
2. **🔵 簡報 (PPT)**：重裝攻擊 (15 dmg)。
3. **🟢 績效考核 (KPI)**：破甲削弱 (0 dmg，但觸發後會使 Boss 進入破甲狀態，下一次承受傷害翻倍 ×2)。觸發時 Boss 血條下方會出現綠色脈衝徽章「下次攻擊 ×2」提示玩家。
4. **🟣 公文 (FILE)**：毒素攻擊 (10 dmg)。
5. **🟠 電話 (PHONE)**：精神干擾/麻痺 (15 dmg，消除 3 顆時 Boss 攻擊倒數 +1，之後每多消除 1 顆再 +1；同一次操作含連鎖最多 +5)。
6. **☕ 續命咖啡 (TEA)**：治癒 (不造成傷害，回復玩家 5 HP)。
7. **⏱️ 摸魚時鐘 (CLOCK)**：延遲 (5 dmg，每消除 1 顆即增加 Boss 攻擊倒數 +1)。
8. **💰 薪水 (MONEY)**：強力增益攻擊 (20 dmg，同時回復玩家 5 HP)。

## 7. 關卡與章節實作現況 (Campaign Implementation)
- **完整戰役**：目前遊戲已於 `levels.js` 中完整實作《Office RPG：社畜大反擊》的全部 6 個章節（從第一章「菜鳥降臨」到第六章「巔峰對決」），總計 85 個關卡。
- **關卡配置**：每個關卡均具備獨特的劇情對話（`start` 與 `win`）、獨立的 Boss 數值（血量、攻擊力、倒數計時）、客製化的 $N \times M$ 盤面佈局（`layout`），以及該關卡允許生成的方塊種類（`allowedBlocks`）。
- **二周目機制**：玩家擊敗最終 Boss 破關後，系統將自動進入「二周目」，重置進度並全面解鎖所有 8 種方塊，讓玩家挑戰最高難度的地獄模式。
- **難度保護機制**：在一周目初期（例如第 1 輪），若關卡配置過多方塊種類，系統會自動減少 1 種方塊，以確保新手玩家能有較順暢的消除體驗。

## 8. 障礙方塊與 Boss 技能 (Obstacles & Boss Skills)
- **卡紙方塊 (obstacle_jam)**：
  - 觸發條件：僅在**二周目（NG+）以上**出現，Boss 每回合有 15% 的機率在盤面隨機空位放置一個卡紙方塊。
  - 效果：無法移動、無法參與三消配對，玩家必須在其**相鄰位置**消除方塊才能破壞卡紙。
  - 視覺呈現：以印表機圖示 (`fa-print`) 搭配斜線條紋背景呈現，強調「卡住」的質感。
- **甩鍋炸彈 (bombTimer)**：
  - 觸發條件：由特定關卡的 Boss 技能在盤面方塊上附加倒數計時。
  - 效果：方塊右上角會顯示倒數數字，每次玩家行動後 -1。當倒數歸零時引爆，對玩家造成 **15 點傷害**，並自動摧毀該方塊。玩家可在倒數歸零前消除該方塊來解除炸彈。

## 9. 動態教學系統 (Dynamic Tutorial System)
- **方塊教學 (Block Tutorials)**：當關卡中出現玩家「從未見過」的方塊種類時，系統會在開場對話結尾自動附加一段教學說明，展示新方塊的圖示與效果描述。已見過的方塊不會重複提示。
- **道具教學 (Prop Tutorials)**：當玩家首次合成出特殊道具（光束、十字、黑洞）或首次觸發道具組合技時，系統會彈出一次性的教學對話框，說明該道具的使用方式。
- **記憶機制**：所有已看過的教學記錄皆儲存於 LocalStorage 中的 `seenTutorials` 與 `seenProps` 欄位，確保跨關卡不重複顯示。

## 10. 遊戲系統架構 (Technical Architecture)
網頁端採用純前端技術開發（HTML5 / CSS3 / JavaScript），確保輕量、免下載、即開即玩。
- **前端 UI 介面**：處理視覺呈現、CSS 動畫與 Web Audio API 音效。
- **遊戲核心邏輯 (MatchEngine)**：負責生成網格矩陣、偵測玩家拖曳事件、判斷三消/四連/五連邏輯，以及執行方塊的掉落與補滿演算法。
- **戰鬥狀態機 (GameState)**：管理核心戰鬥流程：`PLAYER_TURN` (玩家移動) -> `MATCH_PROCESSING` (計算消除與傷害) -> `ENEMY_TURN` (敵人倒數/發動攻擊) -> `CHECK_ALIVE` (判定輸贏)。
- **關卡與劇情資料 (levels.js)**：儲存超過 80 個關卡的 Boss 配置與對話劇本。
- **本地數據儲存 (LocalStorage)**：記錄玩家當前突破的關卡進度與周目，確保重開網頁能延續進度。
