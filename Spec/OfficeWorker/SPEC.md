# Office RPG Match-3 Game Specification

## 1. 遊戲核心機制 (Core Mechanics)
- **基礎消除**：經典三消玩法，滑動相鄰的方塊（員工日常元素）達成三個以上連線即可消除並對 Boss 造成傷害或對玩家產生治癒等效果。
- **Boss 戰鬥系統**：
  - **HP 系統**：玩家與 Boss 皆有生命值，某一方歸零時觸發勝負。
  - **Boss 行動倒數 (Timer)**：玩家每移動一步（或觸發連鎖），Boss 的倒數計時器便會減少。倒數歸零時 Boss 發動攻擊，造成玩家傷害並重置倒數。
- **掉落與重整 (Gravity & Refill)**：
  - 消除方塊後，上方塊受到重力影響自然落下，並從頂部隨機補充新方塊。
  - 這些連續的掉落可能引發後續的連鎖反應（Combo）。
- **死局洗牌 (Shuffle)**：
  - 當畫面上沒有任何可移動消除的組合時，系統將自動洗牌（Shuffle）。
  - 洗牌機制：將「盤面上現有的方塊（包含特殊道具）」打亂並重新排列，而非重新生成全新方塊。系統會嘗試找出不會造成自動連線且至少有一步可移動的排列；若極端情況下多次重排皆無解，才會作為安全機制重新隨機生成。
## 2. 特殊道具與生成規則 (Special Props)
當玩家完成特定的連線形狀時，會將連線轉化為一個強力的特殊道具，並留在盤面上供後續引爆。
1. **水平光束 (↔️ Line Horizontal)**：
   - 生成條件：水平 4 連線。
   - 效果：引爆時清除該方塊所在的**整行**。
2. **垂直光束 (↕️ Line Vertical)**：
   - 生成條件：垂直 4 連線。
   - 效果：引爆時清除該方塊所在的**整列**。
3. **十字炸彈 (💥 Cross Bomb)**：
   - 生成條件：完成 L 型或 T 型的連線。
   - 效果：引爆時清除以自身為中心的一個大區塊。
   - **優化現狀**：生成點已經過嚴格計算，必定精準產生於 L/T 型連線的「交會點（轉角處）」。
4. **彩色黑洞 (🌀 Black Hole)**：
   - 生成條件：完成 5 連線。
   - 效果：與任意顏色方塊交換時，將畫面中所有該顏色的方塊一次全部吸入並消除。
   - **視覺現狀**：圖示為一個旋轉發光的黑洞。消除發動時不使用刺眼反白，而是在每個目標方塊下方產生小黑洞，並跟著方塊同步縮小消失（動畫延遲 500ms，讓玩家看清吸收過程）。

## 3. 道具組合技 (Prop Combos)
當玩家「直接將兩個特殊道具互相交換」時，不再觸發普通的連線消除，而是直接引發威力驚人的組合技：
- **光束 + 光束**：產生十字型光束轟炸，清除一整列與一整行。
- **光束 + 十字**：產生「巨型光束轟炸」，破壞範圍為寬度 3 的十字（清除 3 行 + 3 列）。
- **十字 + 十字**：產生「超大核爆」，範圍擴大至 5x5 的巨大區塊。
- **黑洞 + 一般道具 (光束或十字)**：
  - 效果：會將畫面上所有與「被交換的道具」同色的普通方塊，全部轉化為該道具（例如滿畫面的十字炸彈），然後在一瞬間全部連鎖引爆！
- **黑洞 + 黑洞**：
  - 效果：盤面全清。
  - **視覺現狀**：會在畫面正中央生成一個「覆蓋整個遊戲盤面的超巨大黑洞」，接著整個畫面的方塊會跟著大黑洞一起在 1 秒內坍縮消失，然後才降下新方塊。

## 4. 戰鬥數值與音效表現 (Combat & Audio)
- **連擊加乘 (Combo Multiplier)**：
  - Combo 2：1.2x 傷害/回復
  - Combo 3：1.5x 傷害/回復
  - Combo 4 以上：2.0x 傷害/回復
  - 組合技 (Prop Combos) 觸發時，其造成的巨量傷害也固定享有 2.0x 的終極加成。
  - 多色同時引爆也會再疊加 1.5x 係數。
- **Web Audio 系統**：
  - 揚棄傳統 8-bit 音效，實作了基於 Web Audio API 的進階合成器。
  - 運用 ADSR (Attack, Decay, Sustain, Release) 包絡線、雙振盪器 (Oscillators) 與低通濾波 (Low-pass Filter)，創造出具有 Q 彈感、空間感與和聲的滑順打擊音效。

## 5. 近期技術修正備忘 (Recent Fixes)
1. **API 名稱統一**：修復了 `processPropCombo` 中呼叫到不存在的 `playerAttack` / `stunBoss` 等函式，現已統一對接為 `this.gameState.applyDamage()` 等標準介面。
2. **重力掉落邏輯重構**：將 `processMatches` 中的重力與填充邏輯抽離成獨立的 `applyGravityAndRefill()`，修復了組合技執行後不會掉落方塊的 Bug。
3. **黑洞延遲調校**：黑洞的消除動畫等待時間由 1000ms 縮短至 500ms，在「看清消除過程」與「遊戲流暢節奏」之間取得最佳平衡。
